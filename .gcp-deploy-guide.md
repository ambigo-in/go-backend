# GCP Cloud Run Deployment Guide

This guide helps you deploy the Ambigo Backend to Google Cloud Run.

## Prerequisites

1. **Google Cloud Project**: Create a GCP project at https://console.cloud.google.com
2. **gcloud CLI**: Install from https://cloud.google.com/sdk/docs/install
3. **Docker**: Install Docker Desktop (for local testing)
4. **Authentication**: Run `gcloud auth login` and `gcloud config set project PROJECT_ID`

## Files Created for Deployment

- **Dockerfile**: Multi-stage Docker build configuration
- **.dockerignore**: Files excluded from Docker image
- **.gcloudignore**: Files excluded from Cloud Build
- **cloudbuild.yaml**: Cloud Build pipeline configuration

## Deployment Steps

### Option 1: Using Cloud Build (Recommended for CI/CD)

```bash
# Deploy using Cloud Build (automatically builds and deploys)
gcloud builds submit \
  --config=cloudbuild.yaml \
  --substitutions=_SERVICE_NAME=ambigo-backend,_REGION=us-central1

# View build logs
gcloud builds log <BUILD_ID> --stream
```

### Option 2: Manual Docker Build and Deploy

```bash
# Build locally first (optional, for testing)
docker build -t ambigo-backend .

# Authenticate Docker with GCR
gcloud auth configure-docker

# Build and push to Container Registry
docker build -t gcr.io/PROJECT_ID/ambigo-backend:latest .
docker push gcr.io/PROJECT_ID/ambigo-backend:latest

# Deploy to Cloud Run
gcloud run deploy ambigo-backend \
  --image=gcr.io/PROJECT_ID/ambigo-backend:latest \
  --region=us-central1 \
  --platform=managed \
  --allow-unauthenticated \
  --set-env-vars="MONGODB_URI=<your-mongodb-uri>,JWT_SECRET=<your-secret>,..." \
  --memory=512Mi \
  --cpu=1
```

### Option 3: Deploy Directly from Source (Simplest)

```bash
# Deploy directly from source code
gcloud run deploy ambigo-backend \
  --source=. \
  --region=us-central1 \
  --platform=managed \
  --allow-unauthenticated \
  --set-env-vars="MONGODB_URI=<your-mongodb-uri>,JWT_SECRET=<your-secret>,..." \
  --memory=512Mi \
  --cpu=1
```

## Using Secret Manager

For sensitive credentials, use Google Cloud Secret Manager:

```bash
# Create secrets
echo -n "$MONGODB_URI" | gcloud secrets create mongodb-uri --data-file=-
echo -n "$JWT_SECRET" | gcloud secrets create jwt-secret --data-file=-

# Reference in Cloud Run
gcloud run deploy ambigo-backend \
  --image=gcr.io/PROJECT_ID/ambigo-backend:latest \
  --region=us-central1 \
  --set-env-vars="MONGODB_URI=mongodb-uri" \
  --set-secrets="JWT_SECRET=jwt-secret:latest" \
  ...
```

## Performance Optimization

### Recommended Settings:

```bash
gcloud run deploy ambigo-backend \
  --image=gcr.io/PROJECT_ID/ambigo-backend:latest \
  --region=us-central1 \
  --platform=managed \
  --memory=512Mi \
  --cpu=1 \
  --concurrency=100 \
  --max-instances=100 \
  --min-instances=1
```

**Resource Allocation:**
- **Memory**: 512Mi to 4Gi (start with 512Mi, increase if needed)
- **CPU**: Allocated proportionally (512Mi = 0.25 CPU, 4Gi = 4 CPU)
- **Concurrency**: 50-100 depending on workload
- **Instances**: Set min=1 for always-on, adjust max based on traffic

## Troubleshooting

### Health Check Issues

The Dockerfile includes a health check that expects a `/health` endpoint. Add this to your code if not present:

```go
// In your main.go or router setup
router.HandleFunc("/health", func(w http.ResponseWriter, r *http.Request) {
    json.NewEncoder(w).Encode(map[string]string{"status": "ok"})
}).Methods("GET")
```

### Port Configuration

Cloud Run automatically sets the `PORT` environment variable (default 8080). The application already handles this in `config.go`.

### Container Registry Issues

```bash
# Enable required APIs
gcloud services enable \
  cloudbuild.googleapis.com \
  containerregistry.googleapis.com \
  run.googleapis.com

# View available images
gcloud container images list
```

## Cost Optimization

- Use **Cloud Run** instead of GKE for lower cost
- Set appropriate **min-instances** (0 for auto-scale, 1 for always-on)
- Monitor with **Cloud Monitoring** to right-size resources
- Use **Secret Manager** instead of storing in environment variables
